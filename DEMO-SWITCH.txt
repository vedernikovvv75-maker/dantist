================================================================================
  ПЕРЕКЛЮЧЕНИЕ РОЛЕЙ В ДЕМО-ВЕРСИИ (пациент ↔ заказчик/собственник)
================================================================================

Зачем нужно
-----------
В демо заказчик может войти как «собственник», посмотреть воронку продажи,
потом переключиться и посмотреть интерфейс пациента (/patient). Обратно в режим
заказчика раньше вернуться было нельзя. Добавлено явное переключение в обе стороны,
только для демо-режима, чтобы при переходе на боевой код его можно было легко убрать.


Команды (только при включённом демо-переключателе)
-------------------------------------------------
  /patient  — перейти в режим пациента (главное меню «Улыбка+» с 6 кнопками).
  /owner    — перейти в режим заказчика/собственника (экран воронки продажи).

Кто в какой роли:
  • Сейчас в роли пациента → введите /owner, чтобы снова видеть экран для заказчика.
  • Сейчас в роли заказчика → введите /patient, чтобы посмотреть интерфейс пациента.

В интерфейсе пациента при включённом демо внизу приветствия выводится подсказка:
  «В демо: перейти в режим заказчика — /owner».


Как включено в коде (что убрать для продакшена)
-----------------------------------------------
1. Конфиг (config.py):
   Переменная DEMO_SWITCH_ENABLED (читается из .env: DEMO_SWITCH_ENABLED=true/false).
   По умолчанию true — команды /patient и /owner работают.

2. Регистрация команд (main.py):
   Блок:
     if DEMO_SWITCH_ENABLED:
         app.add_handler(CommandHandler("patient", handle_patient))
         app.add_handler(CommandHandler("owner", handle_owner))
   Пока DEMO_SWITCH_ENABLED=True, обе команды доступны. При False — не регистрируются,
   пользователь не сможет переключать роль по команде.

3. Подсказка в меню пациента (patient_menu.py):
   Если DEMO_SWITCH_ENABLED — к приветствию пациента добавляется строка
   «В демо: перейти в режим заказчика — /owner». При выключенном демо-переключателе
   эта строка не показывается.

4. Обработчики (start.py):
   handle_patient() и handle_owner() — вызываются только когда команды
   зарегистрированы (т.е. при DEMO_SWITCH_ENABLED=True). Для продакшена их можно
   не трогать: достаточно выставить DEMO_SWITCH_ENABLED=false и перезапустить бота.


Что сделать для продакшена (боевой версии)
------------------------------------------
• В .env задать:
    DEMO_SWITCH_ENABLED=false
  либо удалить переменную (тогда по умолчанию в config.py можно сменить на False).

• Перезапустить бота. После этого /patient и /owner не будут обрабатываться,
  подсказка «В демо: перейти в режим заказчика» не появится.

• При желании можно дополнительно в main.py удалить или закомментировать блок
  «if DEMO_SWITCH_ENABLED: ... add_handler(patient) ... add_handler(owner)»
  и в config.py захардкодить DEMO_SWITCH_ENABLED = False — тогда переключатель
  ролей будет полностью отключён одним местом в коде.
