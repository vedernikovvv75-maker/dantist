# Расширенная спецификация: демо-бот «Улыбка+»

Единый референс перед разработкой Telegram-бота для стоматологической клиники.

**Разделение двух режимов: собственник vs пациент**

- При первом `/start` всегда показывается экран собственника (квалификация), как в dantist.
- Если пользователь нажал **«Да, я собственник/маркетолог»** — в БД ставится `is_owner = True`; все последующие `/start` ведут в воронку продажи разработки (чек-лист → демо записи → репутация → напоминалки → трипваер → апселл).
- Если нажал **«Нет, просто смотрю»** или квалификацию не проходил — ведём в режим пациента (главное меню «Улыбка+»).
- **В start.py:** проверка пользователя в БД. Если `is_owner = True` — подключается текущий owner-flow; иначе вызывается `patient_menu.show_main_menu()` с приветствием «Улыбка+». Дублирования `/start` нет, только ветвление по флагу.

## 1. Общая логика демо-бота

### Роль бота

- **Быстро оформить электронную бонусную карту** — лид-магнит, низкий порог входа.
- **Вести долгий прогрев:** напоминания, акции месяца, репутационный контент.
- **Обрабатывать экстренные запросы** (боль, кровотечение, сколы) с круглосуточным маршрутом на живого врача/колл-центр.
- **Подогревать на дорогие услуги:** ортодонтия, имплантация, в т.ч. базальная/одноэтапная.

### Каналы входа

- Кнопка/ссылка на лендинге: «Получить бонусную карту в Telegram».
- Кнопка в рекламе: «Оформить годовую карту в 1 клик».
- QR-код в офлайн-материалах клиники.

## 2. Стартовый сценарий (главное меню)

### Стартовое сообщение

«Здравствуйте! Я — ассистент клиники "Улыбка+". Помогаю оформить электронную бонусную карту, подобрать врача и записаться на удобное время. Чем могу помочь?»

### Кнопки (Reply Keyboard / Inline)

- «Оформить бонусную карту»
- «Записаться на консультацию»
- «У меня острая боль / экстренный случай»
- «Услуги и цены»
- «Наши врачи и отзывы»
- «Уникальная имплантация за 14 дней» (под демонстрацию кейса с базальной имплантацией)

---

## 3. Воронка «Бонусная карта на год»

**Цель:** собрать контакты «на будущее», даже если человеку сейчас не нужен стоматолог.

### 3.1. Оффер

**Сообщение:**

«Электронная бонусная карта:
- скидка до 20% на услуги,
- действует 12 месяцев,
- каждый месяц — спецпредложение,
- оформляется онлайн, без визита.»

**Кнопки:**

- «Получить карту»
- «Узнать детали»

### 3.2. Сбор минимальных данных

**Шаги:**

1. Бот: «Как к вам обращаться?» (Имя).
2. Бот: «Телефон для записи (мы используем его только для напоминаний и уточнения времени)» — кнопка «Отправить мой номер телефона» через Telegram contact.
3. Бот: «Соглашаюсь на обработку данных…» — кнопка «Согласен».

**После этого:**

- Бот генерирует номер карты (или пишет: «Ваша карта активирована»).
- Отправляет: «Готово! Я буду присылать вам напоминания о карте и спецпредложения не чаще 2–3 раз в месяц. Сейчас можно:»

**Кнопки:**

- «Посмотреть акцию месяца»
- «Записаться с картой»
- «Что по моей проблеме?»

## 4. Прогрев: регулярные сценарии сообщений

(В демо — примеры шаблонов рассылок.)

### 4.1. Напоминания о карте

**Пример сообщения:**

«Напоминаю: у вас активна бонусная карта клиники "Улыбка+". До конца месяца — -50% на профессиональную чистку зубов. Записаться сейчас?»

**Кнопки:**

- «Записаться на чистку»
- «Подробнее об акции»
- «Не интересно»

### 4.2. Акции месяца и каталог услуг

**Примеры офферов:**

- «Чистка со скидкой 50% до [дата].»
- «Отбеливание –30% для держателей карты.»
- «Бесплатная консультация по имплантации + 3D-диагностика.»

**Структура:**

- Короткий оффер.
- Фото/картинка (в демо — заглушка).
- Кнопки: «Записаться», «Задать вопрос врачу», «Подробнее о процедуре».

**Каталог:**

- «Гигиена и профилактика»
- «Ортодонтия (брекеты, элайнеры)»
- «Имплантация и протезирование»
- «Детская стоматология»

При выборе направления — 2–3 частых вопроса и кнопка «Подобрать время».

## 5. Репутационный контент в боте

**Цель:** создать эффект «клиника давно на рынке и ей доверяют».

### 5.1. Блок «Наши врачи»

**Структура карточки:**

- Фото врача.
- Имя, специальность.
- Стаж, категории, учёные степени.
- Узкая специализация: «Сложные случаи имплантации», «Исправление прикуса элайнерами».

**Кнопки:**

- «Посмотреть отзывы о докторе» (ссылки на Продокторов и др.).
- «Записаться к этому врачу».

### 5.2. Блок «Отзывы и рейтинги»

**Сценарий:**

- Бот показывает скрин или текстовый фрагмент: «Рейтинг 4,9 на Продокторов».
- Кнопки: «Читать отзывы на сайте», «Смотреть на Продокторов», «Записаться».

### 5.3. Оборудование и технологии

**Короткие карточки:**

- «Лечение под наркозом/седацией».
- «3D-диагностика, микроскоп, современные материалы».

**Кнопки:**

- «Почему это важно?»
- «Хочу консультацию с таким оборудованием».

---

## 6. Экстренные пациенты (круглосуточный сценарий)

**Ветка:** «У меня острая боль / экстренный случай».

### Уточнение типа проблемы (кнопки)

- «Острая боль»
- «Опухоль / воспаление»
- «Скололся зуб/коронка»
- «Кровотечение после удаления»

### Для любой ветки бот

1. Пишет краткий безопасный совет: «Вам необходим очный осмотр стоматолога в ближайшее время.»
2. Предлагает:
   - «Позвонить в клинику сейчас» (кнопка с телефоном).
   - «Отправить фото/описание врачу» (бот принимает медиа и пишет, что администратор свяжется).

### Ночью (демо — условие по времени)

- Сообщение: «Круглосуточная линия экстренной помощи: +7…»
- Кнопка «Попросить перезвонить».

## 7. Ветка «Уникальная имплантация за 14 дней»

Демонстрация кейса с базальной/одноэтапной имплантацией.

### Экран-оффер

«Зубы за 14 дней вместо 6–12 месяцев: одноэтапная имплантация без наращивания кости, без временных протезов.»

**Кнопки:**

- «Подходит ли мне?» (тест)
- «Сколько это стоит?»
- «Записаться на бесплатную консультацию»

### Мини-тест «Подходит ли вам метод»

**Вопросы (кнопки «Да»/«Нет»):**

- «У вас отсутствует несколько зубов подряд?»
- «Вам говорили, что нужна операция по наращиванию кости?»
- «Важен быстрый результат (до 2–3 недель)?»

**В конце:**

- Если много «Да»: «По вашим ответам, метод может вам подойти. Рекомендуем бесплатную консультацию и 3D-диагностику.»  
  Кнопки: «Записаться», «Задать вопрос врачу».

### Работа с возражениями (вопросы-кнопки)

- **«Почему так дорого?»** — аргументация про экономию времени, отсутствие доп. операций, пожизненную гарантию.
- **«Не опасно ли это?»** — краткое описание метода и направление к врачу.
- **«Больно ли?»** — акцент на обезболивание, седацию.

## 8. Ортодонтия как «драйвер» других услуг

**Отдельная ветка:** «Исправление прикуса / брекеты».

### Экран

«Исправьте прикус без переплат — фиксированная цена и рассрочка 0%.»

**Кнопки:**

- «Для взрослого»
- «Для ребёнка»
- «Элайнеры»

### Для каждой ветки

- Короткое описание проблемы и решения.
- Фото/кейсы «до/после» (в демо — заглушки).
- Гарантии: лицензии, опыт врачей.
- Кнопка «Записаться на бесплатную консультацию ортодонта».

### Прогрев по ортодонтии в рассылке

- «Как ухаживать за брекетами?»
- «Мифы об элайнерах.»
- «Сколько реально длится лечение?»

## 9. Техническая структура (упрощённое ТЗ)

### Платформа

- Telegram Bot API + автоматика (n8n/Make/свой backend).
- Готовые примеры сценариев для медботов, связка с календарями и CRM.

### Интеграции

- **CRM** (amoCRM/Битрикс24) — создание лида с тегами: «Бонусная карта», «Экстренный», «Ортодонтия», «Имплантация».
- **Календарь** (Google/Яндекс) — запись на приём.
- **Email/SMS** — дублирующие напоминания.

### Логика статусов

«Новый контакт» → «Получил карту» → «Записан» → «Пациент» → «Повторные визиты».

---

## 10. Как использовать демо-бот в продаже услуги

- Показать карту сценариев: вход из рекламы, оформление карты, шаблоны прогрева, ветки под ортодонтию и имплантацию.
- Подчеркнуть: за счёт бонусной карты и прогрева стоимость лида снижается, жизненный цикл лида удлиняется (человек может прийти через 3–6 месяцев, но уже «тёплым»).
- Показать, как бот собирает данные, создаёт лиды в CRM и помогает измерять ROI и конверсии по каналам.

## Реализация в dantist

### patient_menu: шесть кнопок и маршрутизация

**patient_menu.py**

- Функция `show_main_menu(message)` — отправляет текст из раздела 2 спеки + ReplyKeyboard с кнопками:
  - «Оформить бонусную карту»
  - «Записаться на консультацию»
  - «У меня острая боль / экстренный случай»
  - «Услуги и цены»
  - «Наши врачи и отзывы»
  - «Уникальная имплантация за 14 дней»

**Роутинг по кнопкам:**

- «Оформить бонусную карту» → `bonus_card.start_flow()` (существующий/доработанный лид-магнит).
- «Записаться на консультацию» → `demo_appointment.start_patient_flow()` (тот же сценарий, тексты «для пациента»).
- «У меня острая боль…» → `emergency.start_flow()`.
- «Услуги и цены» → `catalog.show_root()`.
- «Наши врачи и отзывы» → `reputation.show_root()` (в контексте пациента).
- «Уникальная имплантация за 14 дней» → `implant14.start_flow()`.

### Соответствие разделов спеки и handlers

| Раздел спеки | Handler / компонент | Статус |
|--------------|---------------------|--------|
| 2. Стартовый сценарий (главное меню пациента) | start.py + patient_menu (новый) | Бэклог |
| 3. Бонусная карта на год | bonus_card.py | Есть (доработать пост-карту меню) |
| 4. Прогрев, акции, каталог | catalog.py (новый) | Бэклог |
| 5. Репутационный контент | reputation.py | Есть |
| 6. Экстренные пациенты | emergency.py (новый) | Бэклог |
| 7. Имплантация за 14 дней | implant14.py (новый) | Бэклог |
| 8. Ортодонтия | ortho.py или catalog (новый) | Бэклог |
| Демо записи на приём | demo_appointment.py | Есть |

### Бэклог надстройки «Улыбка+»

- **start.py** — проверка `is_owner`; если нет/False — вызов `patient_menu.show_main_menu()`.
- **patient_menu.py** — приветствие «Улыбка+» + клавиатура из 6 кнопок; роутинг по кнопкам в нужные handlers.
- **bonus_card.py** — после активации карты отправлять текст и кнопки: «Посмотреть акцию месяца» / «Записаться с картой» / «Что по моей проблеме?».
- **catalog.py** — разделы услуг; блок «Акция месяца» (шаблон из texts.py + дата); кнопка «Подобрать время» → общий сценарий записи с тегом услуги.
- **emergency.py** — FSM: выбор типа проблемы → опционально медиа; безопасный текст, кнопка звонка, «Отправить фото/описание врачу»; ночной режим (условие по времени) → текст + «Попросить перезвонить».
- **implant14.py** — экран УТП + тест из 3 вопросов Да/Нет; результат (подходит/нужна консультация) + возражения; кнопка «Записаться» → общий сценарий записи с тегом IMPLANT14.
- **ortho.py** (или ветка в catalog) — экран «Исправьте прикус без переплат — рассрочка 0%»; ветки для взрослого / ребёнка / элайнеров → краткий текст + «Записаться на бесплатную консультацию».
