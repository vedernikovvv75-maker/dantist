# Демо-бот «Улыбка+» для стоматологических клиник

Telegram-бот по спецификации [SPEC_ulybka_plus.md](SPEC_ulybka_plus.md): квалификация собственник/пациент, главное меню пациента, бонусная карта, запись на приём, экстренные случаи, каталог услуг, репутация, имплантация за 14 дней, ортодонтия.

## Стек

- Python 3.11+
- python-telegram-bot v20+
- SQLite (файл `bot.db`)
- python-dotenv

## Установка

```bash
cd dantist
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Скопируйте `EnvExample` в `.env` и заполните:

- `BOT_TOKEN` — токен бота от @BotFather
- при необходимости: `ADMIN_ID`, `DATABASE_PATH`, `CLINIC_PHONE`, `SITE_URL`

## Запуск

```bash
python main.py
```

В консоли: **«Демо-бот «Улыбка+» запущен»**.

## Поведение

1. **Первый /start** — экран квалификации: «Да, я собственник/маркетолог» или «Нет, просто смотрю».
2. **Собственник** — в БД `is_owner=1`, при следующих /start показывается заглушка воронки продажи.
3. **Пациент** — главное меню с 6 кнопками:
   - Оформить бонусную карту → воронка: оффер → имя → телефон → согласие → активация → меню «Акция месяца» / «Записаться с картой» / «Что по моей проблеме?»
   - Записаться на консультацию → демо-запись (дата/период → подтверждение)
   - У меня острая боль / экстренный случай → выбор типа → совет → звонок / отправить фото; ночью — круглосуточная линия
   - Услуги и цены → каталог (гигиена, ортодонтия, имплантация, детская) + акция месяца
   - Наши врачи и отзывы → рейтинг, отзывы, карточки врачей
   - Уникальная имплантация за 14 дней → УТП, тест из 3 вопросов, возражения, запись с тегом IMPLANT14

## Структура

| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа, роутеры callback и message |
| `config.py` | load_dotenv, BOT_TOKEN, CLINIC_PHONE, SITE_URL |
| `db.py` | SQLite: users (is_owner, bonus_card, status), funnel_events |
| `texts.py` | Все тексты по спеки |
| `start.py` | Квалификация при /start, ветвление owner/patient |
| `patient_menu.py` | Главное меню пациента, клавиатура, роутинг по кнопкам |
| `bonus_card.py` | Воронка бонусной карты и пост-меню |
| `emergency.py` | FSM экстренных случаев, ночной режим |
| `catalog.py` | Услуги, акция месяца, book_tag → запись |
| `ortho.py` | Ортодонтия: взрослый/ребёнок/элайнеры |
| `reputation.py` | Врачи и отзывы |
| `implant14.py` | Имплантация за 14 дней: тест, возражения |
| `demo_appointment.py` | Демо записи на приём с тегом услуги |

## Статусы в БД

`new` → `got_card` (после активации карты) → `booked` (после заявки на запись) → при необходимости `patient` / `repeat`.

События воронки пишутся в `funnel_events` (qualification_*, bonus_card_*, emergency_*, catalog_*, appointment_* и т.д.) для аналитики и интеграции с CRM.
